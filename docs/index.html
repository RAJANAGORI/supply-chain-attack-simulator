Why SCAS

Software supply chains are the new attack surface

Attackers don't break your firewall first — they break your update path. This simulator gives defenders the tools to test assumptions with real attack patterns.

Real-World Attack Patterns

Based on actual incidents like SolarWinds, CodeCov, event-stream, and XZ Utils. Every scenario maps to documented attack vectors seen in the wild.

Safe by Design

All attacks run in isolated environments with TESTBENCH_MODE gating. Localhost-only operations, clear warning messages, and no real credential harvesting.

Structured Learning Paths

From zero-to-hero guides for beginners to expert-level capstone projects. Teaching packs, module templates, and rubrics included for professional training delivery.

End-to-End Coverage

From dependency resolution abuse to CI/CD compromise, container images, SBOM integrity, and multi-stage chains. Each lab covers attack mechanics, detection signals, and defensive controls.

Built-In Detection Tools

Each scenario includes detection techniques, package scanners, network monitors, and mitigation strategies. Learn to build defenses that catch these attacks before damage.

Team Training Ready

Delivery packs for talks, workshops, and full courses. Designed for red/blue team exercises, SOC training, and DevSecOps skill development across organizations.

How It Works

Six steps from setup to mastery

A structured workflow to maximize learning and validate your organization's supply chain defenses.

1

Define Objective

Choose what you want to validate — CI provenance checks, dependency resolution, or incident response readiness.

2

Select Scenario

Pick from 20 scenarios across beginner to expert levels. Each maps to real-world attack patterns and defensive skills.

3

Run Simulation

Execute the attack in an isolated lab environment with mock infrastructure. All operations are localhost and safety-gated.

4

Observe Telemetry

Monitor captured data, network traffic, and alert quality. Understand how the attack manifests in logs and monitoring tools.

5

Measure Effectiveness

Evaluate MTTD/MTTR and control effectiveness. Quantify gaps in your supply chain security posture with real data.

6

Patch & Revalidate

Fix process gaps identified during testing. Re-run scenarios to validate improvements and measure progress.

Self-Contained Labs

Each scenario has its own setup script, mock attacker server, victim app, and detection tools. No cross-contamination between labs.

CLI-First Design

No dashboard or web UI required. Everything runs from the terminal for maximum transparency and scriptability.

Mock Infrastructure

Node.js-based mock attacker servers capture exfiltrated data locally. Full observability into what the "attacker" receives.

Safety Gated

TESTBENCH_MODE=enabled required to activate payloads. Environment variable checks prevent accidental execution outside labs.

Quick Start

Get running in 5 minutes

Clone, setup, enable testbench mode, and run your first attack scenario.

1

Prerequisites Check

$ node --version # v16+
$ npm --version # v7+
$ git --version
2

Clone & Setup

$ git clone https://github.com/rajanagori/supply-chain-attack-simulator.git
$ cd supply-chain-attack-simulator
$ chmod +x scripts/setup.sh && ./scripts/setup.sh
3

Enable Testbench Mode

$ export TESTBENCH_MODE=enabled
4

Run Your First Scenario

$ cd scenarios/01-typosquatting
$ ./setup.sh
$ node infrastructure/mock-server.js &
$ cd victim-app && npm install ../malicious-packages/request-lib
$ export TESTBENCH_MODE=enabled # required for lab payloads
$ npm start
$ curl http://localhost:3000/captured-data
Defense & Detection
